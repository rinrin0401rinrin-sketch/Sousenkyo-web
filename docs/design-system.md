# Design System

Sousenkyo Web の画面は「選挙特番の即時性」と「Apple Vision Pro風の静かな高級感」を両立させます。Appleロゴ、既存UI、商標表現は使わず、余白、透明感、精密な発光、控えめなアニメーションで表現します。

## Principles

- 白、淡いブルー、薄いグレーを土台にし、政党カラーはデータ表示のアクセントとして使う
- ガラスカード、薄い境界線、柔らかい影、広めの余白を共通ルールにする
- 速報ページでも派手すぎる色や強い点滅を避け、情報の優先度で緊張感を出す
- iPhone幅を先に確認し、見出し、カード、フィルター、ナビが横にはみ出さないようにする
- 政党名、候補者名、選挙区名、議席数はJSON/CSVを正とし、UIへ固定値として増やさない

## Page Roles

- `/`: 現在の選挙、選挙回次、主要導線の入口
- `/live`: 確定、開票中、接戦、未確定を追う速報センター
- `/map`: 小選挙区、比例区、両方表示を切り替える全国結果ページ
- `/parties`: 政党別議席、前回比、小選挙区/比例内訳を見る分析ページ
- `/proportional`: 11ブロック、議席配分、得票率、比例復活を整理するページ
- `/archive`: 過去選挙、今後の選挙、準備中データを同じ枠で扱う入口
- `/glossary`: PDF由来の候補者名、選挙区、比例区分、政党名、用語を軽く確認する単語帳

## Motion

- 初期表示は軽い fade / reveal に留める
- カードの hover は小さな lift と影の変化までにする
- 重要ドットやLive表示は soft pulse にし、激しい点滅は避ける
- `prefers-reduced-motion` ではアニメーションがなくても情報が読める状態を保つ

## Visual Assets

画像は `public/data/{electionId}/visuals/` に置き、設定データから参照します。新しい選挙回次では画像差し替えだけで世界観を維持できるよう、ファイル名、比率、明るさ、余白を揃えます。

画像差し替え後は次を確認します。

```bash
npm run validate:data:strict
npm run build
npm run smoke:preview
```
